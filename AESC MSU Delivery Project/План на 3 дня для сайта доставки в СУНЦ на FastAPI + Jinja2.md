
Этот план рассчитан на быстрый MVP. Цель — успеть собрать **рабочий сайт**, который можно показать и потыкать руками:
- главная страница
- каталог товаров
- оформление заказа
- профиль и история заказов
- страница отслеживания
- чат в упрощённом виде
- 404
- адаптивность для телефона и компьютера

Стек:
- **Backend:** FastAPI
- **Шаблоны:** Jinja2
- **БД:** PostgreSQL
- **Стили:** TailwindCSS или Bootstrap
- **Авторизация:** JWT или простая сессия
- **Деплой:** Docker + Nginx + VPS

---

## Общая стратегия

Сначала делаем **скелет проекта**, потом **логику заказа**, потом **полировку и деплой**.  
Не распыляйся на сложный real-time. Для MVP чат можно сделать через обычные HTTP-запросы и обновление страницы/блока раз в несколько секунд.

# День 1 — Скелет проекта, страницы, БД и авторизация

## Цель дня
Собрать основу проекта так, чтобы сайт уже открывался, были шаблоны, база данных и минимальная авторизация.

## Что сделать

### 1. Создать структуру проекта
Пример структуры:
- `app/main.py`
- `app/models/`
- `app/routes/`
- `app/templates/`
- `app/static/`
- `app/services/`
- `app/db/`
- `app/schemas/`

### 2. Поднять PostgreSQL
Создать таблицы:
- `users`
- `products`
- `orders`
- `order_items`
- `messages`
- `couriers`

На первом этапе достаточно простых полей:
- пользователи: имя, email, пароль, роль
- товары: название, описание, цена, категория, фото
- заказы: пользователь, курьер, статус, итоговая сумма, адрес/точка доставки
- сообщения: автор, заказ, текст, время
- курьеры: имя, статус доступности

### 3. Настроить авторизацию
Минимум:
- регистрация
- вход
- выход
- защита личного кабинета
- разделение ролей: пользователь / курьер / админ

Если времени мало, можно сделать упрощённую сессию вместо сложного JWT, но JWT тоже допустим.

### 4. Сделать базовый layout на Jinja2
Нужны шаблоны:
- `base.html`
- `home.html`
- `catalog.html`
- `profile.html`
- `orders.html`
- `order_create.html`
- `order_track.html`
- `chat.html`
- `about.html`
- `contacts.html`
- `404.html`

### 5. Добавить общий дизайн
Минимум:
- шапка
- меню
- подвал
- блок уведомлений
- карточки
- кнопки
- адаптивная сетка

### 6. Подготовить мок-данные
Если БД ещё не полностью готова, можно временно отрисовать 5–10 тестовых товаров прямо из Python-списка, чтобы уже был видимый результат.

## Результат дня
К концу дня должно работать:
- сайт открывается
- есть страницы
- есть шаблонная структура
- есть база данных
- есть вход/регистрация или хотя бы их каркас
- уже виден каталог с тестовыми товарами

# День 2 — Логика каталога, корзины, заказа и профиля

## Цель дня
Сделать главную пользовательскую ценность: выбрать товар, собрать заказ, отправить его и посмотреть историю.

## Что сделать

### 1. Каталог товаров
Сделать:
- список товаров из БД
- фильтрацию по категориям
- карточки товаров
- кнопку “Добавить в корзину”

Категории:
- обычная еда
- товары из Ozon
- товары из Яндекс Маркета

### 2. Корзина
Минимальная версия:
- хранить выбранные товары в session
- показывать количество и сумму
- удалять товар
- очищать корзину

### 3. Оформление заказа
Страница должна позволять:
- выбрать товары из корзины
- указать точку доставки на территории СУНЦ
- выбрать курьера
- отправить заказ в БД

### 4. История заказов
На странице профиля/истории:
- список прошлых заказов
- статус заказа
- дата
- сумма
- курьер

### 5. Страница отслеживания
Сделать простую страницу, где видно:
- статус заказа
- кто курьер
- время обновления
- последние изменения статуса

### 6. Чат в MVP-формате
Без WebSockets.
Сделать:
- поле для ввода сообщения
- сохранение в БД
- отображение истории сообщений
- обновление через кнопку “Обновить” или polling

### 7. Защита данных
Проверить, что:
- пользователь видит только свои заказы
- чужие заказы не открываются
- чужие сообщения недоступны

## Результат дня
К концу дня должно работать:
- каталог из БД
- корзина
- оформление заказа
- история заказов
- страница трекинга
- простой чат
- ограничения по доступу

## Промпты для нейросетей

## Утро
- открыть вчерашний код
- проверить список задач

## День
- писать только одну фичу за раз
- после каждой фичи запускать проверку в браузере
- не уходить в рефакторинг раньше времени

- устранить баги
- сохранить резервную копию
- выписать, что осталось на завтра
